# File Encryption & Decryption System (Cryptography)

यह प्रोजेक्ट पाइथन के `cryptography` मॉड्यूल (Fernet) का उपयोग करके एक फ़ोल्डर के भीतर की सभी फ़ाइलों को सुरक्षित रूप से एन्क्रिप्ट और डिक्रिप्ट करने के लिए बनाया गया है।

⚠️ **चेतावनी (Warning):** यह प्रोजेक्ट केवल शैक्षणिक उद्देश्यों (Educational Purposes) के लिए है। बिना अनुमति के किसी अन्य के सिस्टम पर इसे चलाना गैर-कानूनी है।

---

## फ़ाइलें और उनका विवरण (Project Structure)

1. **`rw.py`**: यह मुख्य एन्क्रिप्शन स्क्रिप्ट है। इसे चलाने पर यह वर्तमान डायरेक्टरी की सभी फ़ाइलों को एन्क्रिप्ट कर देती है।
2. **`decrypt.py`**: यह डिक्रिप्शन स्क्रिप्ट है। सही गुप्त कोड (Secret Code) दर्ज करने पर यह फ़ाइलों को वापस सामान्य कर देती है।
3. **`secret.key`**: यह फ़ाइल एन्क्रिप्शन के दौरान अपने आप जनरेट होती है। इसके बिना फ़ाइलों को वापस पाना नामुमकिन है।

---

## इसका उपयोग कैसे करें (How to Use)

### 1. फ़ाइलों को एन्क्रिप्ट करना (Encryption)
जब आप `rw.py` स्क्रिप्ट को रन करते हैं:
```bash
python3 rw.py
```
* यह स्क्रिप्ट `rw.py`, `decrypt.py`, और `secret.key` को छोड़कर फ़ोल्डर की बाकी सभी फ़ाइलों को ढूंढती है।
* यह एक सुरक्षित एन्क्रिप्शन की (Key) जनरेट करती है और उसे **`secret.key`** नाम की फ़ाइल में सेव कर देती है।
* यह सभी फ़ाइलों के डेटा को लॉक (Encrypt) कर देती है, जिससे उन्हें बिना डिक्रिप्ट किए पढ़ा नहीं जा सकता।

### 2. फ़ाइलों को डिक्रिप्ट करना (Decryption)
अपनी फ़ाइलों को वापस सामान्य स्थिति में लाने के लिए `decrypt.py` को रन करें:
```bash
python3 decrypt.py
```
* स्क्रिप्ट आपसे गुप्त कोड मांगेगी: `Enter the secret code to decrypt your files`
* आपको सही कोड दर्ज करना होगा: **`Password123`**
* सही कोड मिलने पर यह `secret.key` फ़ाइल को पढ़ेगी और आपकी सभी फ़ाइलों को सुरक्षित रूप से अनलॉक (Decrypt) कर देगी।

## महत्वपूर्ण सुरक्षा निर्देश (Important Notes for secret.key)

* 🔑 **`secret.key` को कभी न खोएं:** यदि आपकी फ़ाइलें एन्क्रिप्टेड हैं और `secret.key` फ़ाइल डिलीट या ओवरराइट (Overwrite) हो जाती है, तो आपका डेटा हमेशा के लिए नष्ट हो जाएगा।
* 🔄 **दोबारा एन्क्रिप्ट न करें:** यदि आप फ़ाइलों के डिक्रिप्ट होने से पहले दोबारा `rw.py` चला देंगे, तो एक नई की (Key) जनरेट हो जाएगी। इससे पुरानी की डिलीट हो जाएगी और पुरानी एन्क्रिप्टेड फ़ाइलें कभी डिक्रिप्ट नहीं हो पाएँगी।
* 🛡️ **सुरक्षित बैकअप:** एन्क्रिप्शन टेस्ट करने से पहले हमेशा अपनी ज़रूरी फ़ाइलों का एक बैकअप बैकअप फ़ोल्डर से बाहर ज़रूर रख लें।
